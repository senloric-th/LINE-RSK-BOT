# PRD — LINE Bot ร้านรักสุขภาพ

## ระบบนี้แก้ปัญหาอะไร · ใครใช้

ร้านรักสุขภาพ (ขายสินค้าสุขภาพ/แม่และเด็ก เช่น ที่ตรวจครรภ์ อาหารเสริมเตรียมตั้งครรภ์) อยากตอบ
ลูกค้าทาง LINE OA ได้ตลอด 24 ชม. โดยไม่ต้องมีแอดมินเฝ้าตลอดเวลา

ผู้ใช้มี 3 ฝั่ง:
- **ลูกค้า** — ทักถามใน LINE OA เรื่องราคา สินค้า การชำระเงิน การจัดส่ง
- **เจ้าของร้าน** — แก้ข้อมูล FAQ/สินค้าเองผ่าน Google Sheet จากมือถือ ไม่ต้องพึ่ง dev
- **แอดมิน** — รับแจ้งเตือนเข้ากลุ่ม LINE เมื่อลูกค้าต้องการคุยกับคนจริง (Smart Handoff)

## คนใช้เข้ามาแล้วทำอะไรได้บ้าง

1. ลูกค้าทักคำถามที่มีคำตอบใน Sheet → บอทตอบในนามแอดมินร้าน ภาษาสุภาพ 1-3 ประโยค
2. ลูกค้าถามด้วยคำที่ต่างจาก Sheet แต่ความหมายเดียวกัน → บอทเข้าใจและตอบถูกเหมือนกัน
3. ลูกค้าถามเรื่องที่ไม่มีข้อมูลใน Sheet → บอทตอบ "ทางร้านยังไม่มีข้อมูลส่วนนี้ค่ะ ขออภัยด้วยค่ะ"
4. ลูกค้าขอเปรียบเทียบสินค้า ("ตัวไหนดีกว่ากัน"), ขอคุยกับแอดมิน, หรือร้องเรียน →
   บอทตอบขอให้แอดมินติดต่อกลับ + ส่งข้อความแจ้งเข้ากลุ่มแอดมินทันที (ถ้าตั้ง `ADMIN_GROUP_ID` ไว้)
5. เจ้าของร้านเพิ่ม/แก้ข้อมูลใน Google Sheet → บอทเห็นข้อมูลใหม่ภายใน 60 วินาที ไม่ต้อง deploy ใหม่

## ต้องเก็บข้อมูลอะไรบ้าง

- FAQ/ข้อมูลสินค้า: หมวด, คำถาม, คำตอบ (ใน Google Sheet ของเจ้าของร้าน ไม่ใช่ฐานข้อมูลแยก)
- ไม่เก็บประวัติแชทลูกค้าฝั่งเรา (LINE เก็บประวัติแชทของตัวเองอยู่แล้ว)

## ใครเห็นข้อมูลที่เก็บ · เห็นยังไง

- เจ้าของร้านแก้/ดู FAQ ได้ตรงผ่าน Google Sheet เดิม (ไม่มีหน้า admin แยกในรอบนี้)
- แอดมินเห็นเคส handoff ผ่านข้อความที่บอทส่งเข้ากลุ่ม LINE ที่ตั้งไว้

## หน้าตา/โทน เป็นแบบไหน

ไม่มีหน้าเว็บสำหรับลูกค้า (เป็น LINE bot ล้วน) โทนการตอบ: สุภาพ เป็นกันเอง เหมือนแอดมินร้าน
คุยกับลูกค้าจริง ไม่ใช้ Markdown/อีโมจิเยอะ ไม่มีลูกเล่นเร่งขาย

## รอบแรกทำแค่ไหน · อะไรเก็บไว้ทำทีหลัง

**รอบแรก (ทำแล้ว):** webhook + signature verify, FAQ จาก Sheet (ไทย/อังกฤษ) + cache 60s +
fallback เมื่อดึงไม่ได้, Gemini ตอบตาม FAQ เท่านั้น (ห้ามแต่งราคา/สรรพคุณ/เปรียบเทียบสินค้า),
Smart Handoff (keyword + Gemini เข้าใจเจตนา) แจ้งแอดมินผ่าน LINE group, retry ตอนส่งข้อความกลับ
LINE, structured logging

**เก็บไว้ทำทีหลัง (ยังไม่ทำรอบนี้ — รอข้อมูล/การตัดสินใจเพิ่มจากเจ้าของร้าน):**
- Flex Message การ์ดสินค้า/ราคาแบบรูปภาพ (ต้องมีรูปสินค้าจริง)
- Flex Message การ์ด QR พร้อมเพย์ (ต้องมี QR/เลขบัญชีจริงแบบรูป)
- Flex Message การ์ดข้อมูลติดต่อร้าน (เบอร์/แผนที่)
- Rich Menu (ต้องออกแบบกราฟิก 2500×1686px ก่อน)
- ระบบกรองข้อมูล FAQ ก่อนส่งเข้า Gemini (ต้องทำถ้า FAQ รวมกันเกิน ~20,000 ตัวอักษรในอนาคต)
- ชุดทดสอบบทสนทนา (test conversation library) แบบเป็นทางการ
